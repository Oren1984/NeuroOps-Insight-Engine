# n8n Automation Integration

This folder contains the automation integration layer for the **AI Business Intelligence Platform**.

The purpose of this component is to simulate how insights generated by the system can be forwarded to external automation workflows such as notifications, incident management, CRM updates, or reporting pipelines.

In this project the integration is implemented in **demo mode**, allowing the automation layer to exist without requiring external infrastructure.

---

## Purpose

The n8n integration demonstrates how the platform could trigger automation based on business intelligence results.

Examples of automation scenarios include:

• sending executive summaries to management  
• creating tickets for repeated system failures  
• notifying product teams about inactive users  
• pushing insights to Slack, email, or monitoring systems  

The current project uses a **webhook-based architecture** where the AI system generates a structured payload and forwards it to an automation engine.

---

## Files

### demo_bi_webhook_workflow.json

This file contains a simple example workflow for n8n.

The workflow includes:

1. **Webhook Node**

Receives a POST request from the AI BI platform.

2. **Transform Payload Node**

Processes the incoming JSON payload and extracts important fields such as:

- event type
- insight count
- executive summary

3. **Respond to Webhook Node**

Returns a response to the sender confirming the payload was received.

---

## Example Payload

The AI system sends a payload similar to:

```json
{
  "event": "ai_bi_summary",
  "timestamp": "2026-03-15T18:00:00Z",
  "source": "ai-business-intelligence-agent",
  "insights": [...],
  "executive_summary": "System health is stable but repeated login errors require investigation."
}
```

## Demo Mode

By default the project runs in demo mode.

This means:

- no webhook is required
- no automation is triggered
- payloads are simply previewed in the UI

---

## Environment Configuration

N8N_ENABLED=false
Enabling Live Automation

To enable real automation:

1. Start n8n using Docker.

2. Import the workflow file:

n8n/demo_bi_webhook_workflow.json

3. Enable the webhook in n8n.

4. Update the .env file:

N8N_ENABLED=true
N8N_WEBHOOK_URL=http://localhost:5678/webhook/ai-bi-summary

After enabling this, the Automation tab in the Streamlit UI will send real payloads to n8n.

## Architectural Role

Within the platform architecture, n8n acts as the automation layer.

AI Analytics
      ↓
Insights Engine
      ↓
Executive Summary
      ↓
Automation Trigger
      ↓
n8n Workflow
      ↓
External Systems

This separation ensures that the AI system remains focused on analysis and reasoning, while automation platforms handle workflow orchestration.

---

## Summary

The n8n integration demonstrates how the AI Business Intelligence Platform can connect insights with automated operational workflows without embedding automation logic directly into the core analytics system.
